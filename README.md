# silhouette-stl-maker

学校教材として使いやすい、シンプルなシルエットを3Dプリンタ用STLに変換する静的Webアプリです。ユーザーが「ハート」「星」「本」「ひまわり」などの登録済みキーワードを入力すると、プロジェクト内のオリジナルSVGシルエットをブラウザ内で押し出し、STLファイルとして保存できます。

## できること

- 日本語・英語・一部記号のキーワードから登録済みシルエットを検索できます。
- 20種類以上の自作SVGシルエットを候補一覧から選べます。
- 幅・高さ・厚みをmm単位で変更できます。
- キーホルダー用の丸いタブと穴を追加できます。
- Three.jsの3DプレビューをOrbitControlsで回転・ズームできます。
- STLExporterでブラウザ内だけでSTLをダウンロードできます。

## できないこと

- AIで画像を生成する機能はありません。
- すべての言葉をSTL化できるわけではありません。
- 登録済みのアイコンのみ対応します。
- 複雑な写真やイラストをSTL化する機能はありません。
- 商標・キャラクター・ロゴのSTL化には対応しません。
- 外部画像検索、任意画像アップロード、サーバー処理、ログイン、広告、トラッキングはありません。

## 使い方

1. キーワード入力欄に「ハート」「星」「本」「ひまわり」などを入力します。
2. 候補一覧から形を直接選ぶこともできます。
3. 幅・高さ・厚みを設定します。初期値は50mm × 50mm × 3mmです。
4. 必要に応じてキーホルダー用タブ、穴直径、ベベルを設定します。
5. 2D/3Dプレビューと警告を確認します。
6. 「STLをダウンロード」を押して保存します。

## ローカル実行方法

```bash
npm install
npm run dev
```

ビルド確認:

```bash
npm run build
```

## GitHub Pages公開方法

`vite.config.ts` は `base: './'` にしているため、ビルド後の `dist` をGitHub Pagesに配置しやすい設定です。

このアプリはVite/TypeScriptアプリなので、リポジトリ直下をそのままGitHub Pagesで公開しても動きません。必ず `npm run build` で生成した `dist/` を公開してください。

推奨手順:

1. GitHubの Settings → Pages → Build and deployment で Source を **GitHub Actions** にします。
2. `main` ブランチへpushすると、`.github/workflows/pages.yml` が `npm install` と `npm run build` を実行します。
3. 成功すると、生成された `dist/` がGitHub Pagesへ公開されます。

手動で公開する場合:

1. 手元で `npm install` を実行します。
2. `npm run build` を実行します。
3. 生成された `dist/` の中身をGitHub Pagesの公開対象にします。

## アイコン追加方法

`src/data/iconRegistry.ts` に `IconDefinition` を追加します。各アイコンには必ず `id`, `label`, `aliases`, `svg`, `license`, `source`, `modified`, `printability` を設定してください。初期アイコンは `viewBox="0 0 100 100"` の単純な塗りつぶしSVGとして作っています。

外部アイコンを追加する場合は、必ず `THIRD_PARTY_LICENSES.md` に出典、ライセンス、改変の有無を記録してください。ライセンス不明の画像は使用しないでください。

## ライセンス方針

- このプロジェクトの初期SVGアイコンは、このプロジェクト用に作成された単純なオリジナルシルエットです。
- ライセンス不明の画像は使用していません。
- 著名キャラクター、企業ロゴ、ブランドロゴは含めていません。
- 外部アイコンを追加する場合は、`THIRD_PARTY_LICENSES.md` に出典・ライセンス・改変の有無を必ず記録してください。
- npm依存関係は `package.json` を参照してください。

## 教材利用時の注意

- 対応していない言葉を入力した場合は、候補一覧から形を選んでください。
- 生徒が商標・キャラクター・学校ロゴなどの形を無断で作らないように説明してください。
- STLは授業用のMVPとして生成しています。重要な造形ではスライサーで必ず確認してください。

## 3Dプリンタ出力時の注意

- 厚みが1.5mm未満だと折れやすくなります。
- 30mm未満の小さいモデルは細部がつぶれることがあります。
- 星、花、ひまわり、音符などは細い部分があるため、印刷後に折れないか確認してください。
- キーホルダー穴の周囲は2mm以上の肉厚を推奨します。
